Create `backend/services/ai_coordinator.py` with the following structure. All comments in Russian per CLAUDE.md rules.

**Module structure:**

1. **Imports:**
   - `json`, `re` from stdlib
   - `anthropic` (for error types: `RateLimitError`, `APIConnectionError`, `APIStatusError`)
   - `AsyncAnthropic` from `anthropic`
   - `structlog` for logging (existing pattern in other services)
   - `from config import settings`
   - `from schemas.coordinator import AgentInfo, AgentCall, QualityEvaluation, CoordinatorError`

2. **Constants:**
   - `CLAUDE_MODEL = "claude-sonnet-4-6"` (locked by project decision)
   - `log = structlog.get_logger(__name__)`

3. **Module-level client:**
   - `_client = AsyncAnthropic(api_key=settings.ANTHROPIC_API_KEY)`
   - Note: tests patch `services.ai_coordinator._client` (run from `backend/`), so this name is mandatory

4. **Helper `_parse_json_response(text: str) -> dict | list`:**
   - Try `json.loads(text.strip())` first
   - Fallback: regex `r"```(?:json)?\s*([\s\S]+?)\s*```"` to extract from markdown code fence
   - If both fail: `raise ValueError(f"Не удалось распарсить JSON: {text[:200]}")`

5. **`async def route_task(task: str, available_agents: list[AgentInfo]) -> list[AgentCall]`:**
   - Build `agent_list` string: each agent as `"- {slug}: {description} | capabilities: {capabilities}"`
   - System prompt: `"Respond ONLY with valid JSON. No explanation, no markdown."`
   - User prompt (in Russian): includes agent list and task, asks for JSON array `[{"slug": "...", "input": {...}, "reason": "..."}]`
   - Call `_client.messages.create(model=CLAUDE_MODEL, max_tokens=1024, system=..., messages=[...])`
   - Parse response with `_parse_json_response(response.content[0].text)`
   - Filter returned slugs: only keep those present in `{a.slug for a in available_agents}`
   - Validate each item as `AgentCall(**item)`
   - Log with structlog: `log.info("route_task_done", task_preview=task[:50], agent_count=len(calls))`
   - Wrap Claude API errors in CoordinatorError (see error handling below)
   - Return `list[AgentCall]`

6. **`async def evaluate_output(agent: AgentInfo, input_data: dict, output_data: dict, execution_id: str) -> QualityEvaluation`:**
   - System prompt: `"Respond ONLY with valid JSON. No explanation, no markdown."`
   - User prompt (in Russian): agent info, input_data as JSON, output_data as JSON, asks for `{"score": 0-100, "reasoning": "..."}`
   - Call `_client.messages.create(model=CLAUDE_MODEL, max_tokens=512, system=..., messages=[...])`
   - Parse with `_parse_json_response`
   - **Clamp score:** `score = max(0, min(100, int(data["score"])))` — critical for u8-safe on-chain storage (REQ-09)
   - **Decision logic:** `decision = "complete" if score >= settings.AI_QUALITY_THRESHOLD else "refund"` (REQ-08)
   - Create `QualityEvaluation(score=score, reasoning=data.get("reasoning", ""), decision=decision, execution_id=execution_id)`
   - Log: `log.info("evaluate_output_done", execution_id=execution_id, score=score, decision=decision)`
   - Return `QualityEvaluation`

7. **Error handling (wrap all Claude API calls):**
   - `except anthropic.RateLimitError` -> `log.warning(...)` -> `raise CoordinatorError("Claude rate limit")`
   - `except anthropic.APIConnectionError` -> `log.error(...)` -> `raise CoordinatorError("Cannot reach Claude API")`
   - `except anthropic.APIStatusError as e` -> `log.error(...)` -> `raise CoordinatorError(f"Claude API error {e.status_code}")`
   - `except (ValueError, json.JSONDecodeError, KeyError) as e` -> `log.error("json_parse_error", ...)` -> `raise CoordinatorError(f"Ошибка парсинга ответа Claude: {e}")`

**Important anti-patterns to AVOID:**
- Do NOT call Solana/Anchor instructions from this module (Phase 3 scope)
- Do NOT use `Anthropic()` sync client — must be `AsyncAnthropic()`
- Do NOT hardcode the quality threshold — use `settings.AI_QUALITY_THRESHOLD`
- Do NOT raise exceptions for score < 70 — it's a valid business outcome (refund), not an error

**Celery/asyncio pitfall (Phase 3 concern):** The module-level `_client = AsyncAnthropic(...)` binds to the current event loop at import time. When called from Celery's `asyncio.run()` (which creates a new event loop per task), the bound httpx client may throw `RuntimeError: Event loop is closed`. Phase 3 wiring must handle this by creating a fresh `AsyncAnthropic` instance inside the Celery task's async scope, or by using `anyio` as the Celery worker. This is documented here so Phase 3 planner is aware. Phase 2 tests are not affected (they mock `_client` entirely).
